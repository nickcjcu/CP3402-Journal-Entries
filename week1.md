# Week 1

## Learning Activities & Resources
This week, I focused on revising foundational web development concepts and preparing to create a simple static website for my startup, ActionBoost. I revisited how HTML is used to structure webpages, including creating headings, paragraphs, lists, images, and links. I also practised using CSS to control layout, colours, fonts, and spacing to make the pages visually appealing and easy to read.  

To make the website more responsive and beginner-friendly, I used Bootstrap as a framework. I explored Bootstrap templates and followed the guide to understand basic components, containers, and responsive design. Using Bootstrap helped me structure the site quickly without needing to write complex CSS from scratch. I created a homepage, an about us page, a features page, and a contact page, ensuring each page contained enough information for a visitor to understand the purpose of the startup. I also practised linking these pages together using a navigation bar so users could move around the site easily.  

To host my site online, I deployed it using GitHub Pages, following instructions for publishing a static site. This allowed me to make the site publicly accessible and practise using version control with GitHub.  

To support my learning, I watched beginner-friendly YouTube tutorials, read articles on HTML, CSS, and Bootstrap, and followed GitHub Pages documentation to successfully deploy the website. Practising coding small sections of the site while referencing these materials helped me consolidate previously learned concepts and apply them in a practical context.

## Content Insights
This week allowed me to revisit fundamental web development practices and apply them in a structured project. Creating multiple interconnected pages helped me understand the importance of site architecture and navigation flow, as even a simple website needs clear pathways for users to access information. Experimenting with layout and spacing reinforced how design choices impact readability and user experience, and using Bootstrap highlighted how frameworks can streamline consistent styling across a site while maintaining flexibility for content presentation.

I learned how to organise content logically, decide what information should appear on each page, and ensure that the design supports clear communication of the website’s purpose. Experimenting with Bootstrap helped me edit responsive layouts quickly using containers, spacing classes, and basic components, which made the site look structured and professional. Deploying the site with GitHub Pages gave me practical experience with hosting and version control, showing how websites are published and maintained online. This project connects theory and practice to demonstrated the full workflow of building a small static website, from planning and content placement to styling and deployment.

## Career/Employability/Learning Insights
Revising foundational web development concepts is important for employability because it ensures that I have a solid understanding of how websites are built and structured. Being confident with HTML, CSS, Bootstrap, and basic layout techniques is a skill that is relevant for any web development, frontend, or IT role.  

This project reinforced several key professional skills: problem-solving when figuring out how to structure pages and link content, adaptability in learning Bootstrap for the first time, and attention to detail in ensuring pages are readable and navigable. Deploying the site online also gave me insight into real-world workflows involving GitHub and version control, which are highly valued in industry. I learned that a combination of guided tutorials, reading official documentation, and hands-on experimentation is the most effective way to learn new web technologies and that these learning strategies are directly transferable to workplace projects and continuous professional development.

# Week 4

## Learning Activities & Resources
This week, I focused on learning how to set up a local development environment and run a dynamic website with a database on my own computer. I researched different tools for local development such as XAMPP, Docker, and LocalWP. After comparing them, I chose LocalWP because it provides a simpler setup and is designed specifically for WordPress.

I followed tutorials to understand how local environments work and how servers, databases, and websites connect together. I learned how to create a new local WordPress site using LocalWP, which automatically sets up PHP, MySQL, and a local server. I explored how the site runs on a localhost URL and how it differs from a live hosted site.

I also looked into how deployment works, including the idea of moving a website from a local environment to a live server. This included understanding the differences between development and production environments, and why testing locally is important before publishing changes.

## Content Insights
A local development environment allows a website to run on your own computer instead of being hosted online. This means you can build and test features without affecting a live site. LocalWP simplifies this process by automatically setting up the required components such as a web server, PHP, and a database.

I learned that dynamic websites rely on a database to store content such as pages, posts, and settings. WordPress connects to this database to display content on the site. Running this locally helped me understand how the backend and frontend are connected.

I also learned the difference between local and live environments. A local environment is used for development and testing, while a production environment is what users access online. This is important because changes should be tested locally before being deployed to avoid breaking the live site.

## Career/Employability/Learning Insights
Understanding local development is an important skill in web development. It allows developers to work more efficiently, test changes safely, and fix issues before deployment. This is a standard workflow used in real-world development environments.

Learning tools like LocalWP also shows the importance of choosing the right tools for the task. Simpler tools can speed up development and reduce setup time, which is valuable in professional settings.

This week improved my understanding of how websites work behind the scenes, especially the role of servers and databases. It also reinforced my learning approach of researching first, then applying the concepts عملي, and then reviewing to strengthen understanding.

## Content Insights
A child theme allows you to customise a WordPress theme without modifying the original parent theme. This is important because any changes made directly to the parent theme can be lost when the theme updates. A child theme solves this by keeping custom changes separate.

I learned that the `style.css` file is used to define the child theme and link it to the parent theme using the Template field. The `functions.php` file is used to properly load the parent and child styles. This setup ensures that the original design is preserved while allowing custom changes.

Making visual changes through the child theme helped me understand how themes control the appearance of a website. Small CSS changes can significantly affect layout and design. This showed me how front-end customisation works in a real CMS environment.
